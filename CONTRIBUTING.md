# Contributing to Aether

Thank you for your interest in contributing to Aether! We welcome contributions and collaborations from developers, researchers, students, and mental health professionals who share our vision for a privacy-first, research-driven student resilience platform.

## How to Contribute

- **Report Bugs:** If you find a bug or unexpected behavior, please open an issue with clear steps to reproduce.
- **Suggest Features:** Have an idea for a new feature or improvement? Open an issue or start a discussion.
- **Submit Pull Requests:**
  1. Fork the repository and create your branch from `main`.
  2. Make your changes with clear, descriptive commit messages.
  3. Ensure your code passes all tests and adheres to our style guidelines.
  4. Submit a pull request and describe your changes in detail.
- **Improve Documentation:** Help us keep our docs clear, up-to-date, and accessible.
- **Accessibility & Ethics:** We especially welcome contributions that improve accessibility, privacy, and ethical AI compliance.

## Collaboration

We encourage collaborations with:
- **Universities & Research Groups:** For pilot studies, research validation, and new feature co-development.
- **Mental Health Professionals:** To ensure interventions are evidence-based and culturally responsive.
- **Open Source Contributors:** To help us build a robust, scalable, and inclusive platform.

If you are interested in a formal collaboration, please contact the project owner via GitHub or open a discussion.

## Code of Conduct

We are committed to fostering a welcoming and inclusive environment. Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Getting Started

1. Fork the repo and clone it locally.
2. Install dependencies: `npm install`
3. Start the dev server: `npm run dev`
4. Make your changes and submit a pull request!
